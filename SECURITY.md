## 报告安全问题

如果你发现安全问题，请不要把可利用细节直接公开在普通 Issue 里。

可以先创建一个不包含敏感细节的 Issue，说明“发现安全问题，需要私下沟通”，或者通过 GitHub 的安全报告功能联系项目维护者。

## 下载安全建议

- 建议只从本仓库 Release 页面下载完整包。
- 下载分卷后运行 `merge-full-package.ps1`，脚本会校验 SHA256。
- 不建议从第三方网盘或未知来源下载改包版本。
- 源码仓库不包含 `.venv` 和模型权重；完整运行环境只放在 GitHub Release 附件中。
